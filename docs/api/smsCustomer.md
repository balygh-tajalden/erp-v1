# 📖 دليل المطوِّر: واجهة برمجة التطبيقات (API) لنظام رسائل الموبايل (SMS Gateway)

مرحباً بك في دليل التوثيق الرسمي لـ API نظام بوابات الرسائل.
يوفر هذا الدليل التفاصيل الكاملة للمطورين (تطبيقات الموبايل، الويب، سطح المكتب) حول كيفية ربط واجهات تسجيل الدخول وإدارة تراخيص الأجهزة مع السيرفر المركزي.

---

## 🔗 الإعدادات العامة (General Configurations)
- **Base URL:** `https://[your-domain.com]/api`
- **Headers:** يجب إرفاق الترويسات التالية مع جميع الطلبات (Requests):
  - `Accept: application/json`
  - `Content-Type: application/json`

---

## 1. مصادقة الدخول والتحقق من حالة الجهاز (Check Status)

يتم استخدام هذا المسار لمحاولة تسجيل دخول العميل باستخدام بياناته، والتحقق في نفس الوقت مما إذا كان جهازه الحالي مرخصاً ومقبولاً من قبل إدارة النظام.

- **الرابط:** `/clients/check-status`
- **نوع الطلب:** `POST`

### 📥 البيانات المرسلة (Request Body)
يجب إرسال البيانات التالية بنسق `JSON`:
```json
{
    "username": "client_username",        // (إجباري) اسم المستخدم الخاص بالعميل
    "password": "client_password",        // (إجباري) كلمة المرور
    "subscription_key": "unique_device_id" // (إجباري) المعرّف الفريد والثابت للجهاز (مثل Device ID أو MAC)
}
```

### 📤 الردود المتوقعة والتصرف البرمجي (Responses & Handling)

| كود HTTP | معرف الخطأ (error_code) | وصف الحالة (متى يحدث؟) | الإجراء المطلوب من المطور (Client) |
| :---: | :--- | :--- | :--- |
| **200** | `(لا يوجد)` | **(نجاح):** بيانات الدخول صحيحة والجهاز مرخص ومفعل. | الانتقال فوراً إلى لوحة التحكم (Dashboard). يتم إرجاع حقل `balance` بالرصيد. |
| **401** | `INVALID_CREDENTIALS` | اسم المستخدم أو كلمة المرور خاطئة. | إظهار رسالة تفيد بخطأ في بيانات الدخول. |
| **403** | `ACCOUNT_SUSPENDED` | حساب العميل بالكامل محظور أو موقوف إدارياً. | إظهار رسالة تفيد بإيقاف الحساب وعدم السماح بأي إجراء. |
| **401** | `DEVICE_NOT_REGISTERED` | الدخول صحيح، ولكن **الجهاز جديد** ولم يتم تسجيله من قبل. | **هام:** إظهار شاشة/حقل جديد للمستخدم يطلب منه إرسال "طلب ترخيص الجهاز". (استدعاء المسار الثاني ⏬) |
| **403** | `DEVICE_PENDING` | الدخول صحيح والجهاز مسجل، ولكنه **قيد المراجعة** من قبل الإدارة. | إظهار رسالة: "جهازك قيد المراجعة، يرجى الانتظار لحين الموافقة". |
| **403** | `DEVICE_REJECTED` | الدخول صحيح والجهاز مسجل، ولكن الإدارة **رفضت** ترخيصه. | إظهار رسالة: "تم رفض ترخيص هذا الجهاز مسبقاً من الإدارة". |

## 2. طلب ترخيص جهاز جديد (Request License)

يتم استدعاء هذا المسار **فقط** عندما يتلقى المبرمج الرد `DEVICE_NOT_REGISTERED` من المسار الأول. وظيفة هذا المسار هي رفع تفاصيل هذا الجهاز الجديد للإدارة للموافقة عليه.

- **الرابط:** `/clients/request-license`
- **نوع الطلب:** `POST`

### 📥 البيانات المرسلة (Request Body)
يجب إرسال البيانات التالية بنسق `JSON`:
```json
{
    "username": "client_username",        // (إجباري) اسم المستخدم للعميل
    "password": "client_password",        // (إجباري) كلمة المرور 
    "subscription_key": "unique_device_id", // (إجباري) المعرّف الفريد للجهاز
    "device_name": "Galaxy S23 أو Win 11"   // (اختياري) اسم الجهاز أو الموديل ليسهل على الإدارة التعرف عليه
}
```

### 📤 الردود المتوقعة والتصرف البرمجي (Responses & Handling)

| كود HTTP | معرف الخطأ (error_code) | وصف الحالة (متى يحدث؟) | الإجراء المطلوب من المطور (Client) |
| :---: | :--- | :--- | :--- |
| **200** | `(لا يوجد)` | **(نجاح):** تم رفع طلب الترخيص بنجاح وهو الآن بانتظار موافقة الإدارة. | إخفاء زر التقديم وإظهار رسالة: "تم رفع الطلب بنجاح وهو قيد المراجعة". |
| **401** | `INVALID_CREDENTIALS` | بيانات المستخدم غير صحيحة، أو تم تغييرها. | إعادة توجيه المستخدم لحقل إدخال الباسوورد. |
| **403** | `MAX_DEVICES_REACHED` | العميل استنفذ الحد الأقصى للأجهزة المسموح له بربطها (مثلاً 3 أجهزة). | إظهار رسالة: "عفواً، لقد وصلت للحد الأقصى للأجهزة المسموحة لك". |
| **403** | `DEVICE_ALREADY_ACTIVE` | حدث خطأ من التطبيق وقام بإرسال طلب لجهاز مفعّل أصلاً. | توجيه المستخدم مباشرة للوحة التحكم. |

---

## 💡 نصائح هامة لمبرمج الواجهات (Front-end / Mobile App)
1. **الاعتماد على `error_code`:** يُرجى عدم الاعتماد على كود الشبكة `HTTP 401/403` فقط، بل يجب قراءة حقل الـ `error_code` المرفق مع الرد لتحديد الإجراء المناسب والشاشة المناسبة للمستخدم.
2. **رسائل المستخدم (message):** جميع الردود تتضمن حقلاً يسمى `"message"` يحتوي على رسالة توضيحية باللغة العربية جاهزة للعرض للمستخدم النهائي (End-User) بداخل رسائل التنبيه (Toast أو Alert).
